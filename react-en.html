Setting up a Docker Environment

To facilitate the development and deployment of my React applications, I have created a Docker environment (with Docker-Compose). The Docker environment allows for easy integration of the backend and frontend of applications, as well as the necessary tools for continuous integration and deployment (Jenkins).

  • Install Docker and Docker-compose on the server (Linux Debian)
  • Create the Jenkins and MongoDB Dockers via DockerHub (Pull ready-to-use images)
  • Create the Node Dockerfiles for the backend and frontend (Build images based on Alpine)

Setting up Github Repository

I generated an access token for my Github repository to allow Jenkins to function properly. This helps restrict access to the application's source code. I then configured this token as a "secret" in Jenkins, allowing Jenkins to access the application's source code on Github, build it, and deploy it automatically.

Token Github
Setting up Jenkins

To set up Jenkins, I created pipelines based on configuration files called Jenkinsfiles. These pipelines define the steps to execute for application integration and deployment. In the near future, I plan to implement tools to control the quality of the application's source code (such as SonarQube). The "environment" part is now in place, and the process is automated and reliable for the development and deployment of my React applications.

Preparing the Database

To prepare the database for the React application, I installed and configured MongoDB, created a database and collections, and integrated MongoDB into my application. I also set up regular backups to ensure data security.

MongoDB collection
Application Development

The application was developed using the React framework for frontend development and Node.js for backend development. I created React components for the different pages of the application and connected them to the backend API via HTTP requests. For backend development, I created endpoints for the different application functionalities and used MongoDB for data storage. I also implemented functions for data validation, authentication, and error handling.
